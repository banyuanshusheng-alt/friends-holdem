# ♠ 仲間内ホールデム

仲間内で遊ぶ、オンライン対戦テキサスホールデム・ポーカー。
ルームコードで集まって、各自のスマホ／PCからすぐに対戦できます。

## できること

- **本格ゲーム進行（全自動）**：配札 → ブラインド → プリフロップ → フロップ → ターン → リバー → ショーダウンまで自動。役判定・勝者判定・チップ移動・**サイドポット／オールイン**もすべてサーバーが自動処理します。
- **オンライン対戦**：各自の端末から接続。自分の手札は自分だけに見え、ショーダウンで残った人の手札だけが公開されます。
- **ルームコード方式**：アカウント登録不要。名前を入れて4文字コードで入室。
- **設定変更**：部屋作成時に初期チップ・SB・BB を自由に設定（デフォルト 1000／10／20）。
- **成績の可視化**：セッション累計の **順位表** と **チップ推移グラフ** をいつでも確認。**勝敗推移はワンタップでリセット** 可能。
- **リバイ**：チップ切れのプレイヤーに、ホストがチップを追加できます。
- **再接続**：リロードや通信切断があっても、同じ端末なら元の席・チップに復帰します。

## ファイル構成

```
ポーカー/
├── server.js              … Express + Socket.IO サーバー（静的配信＋リアルタイム対戦）
├── src/
│   ├── rooms.js           … ルームの生成・検索・掃除
│   ├── room.js            … 1部屋の状態（プレイヤー・チップ・成績・ハンド進行）
│   └── poker/
│       ├── deck.js        … カード定義・シャッフル
│       ├── handEvaluator.js … 7枚から最強5枚役を判定
│       └── game.js        … 1ハンドの進行（ベット・ポット・サイドポット・配当）
├── public/                … フロントエンド（ビルド不要）
│   ├── index.html
│   ├── styles.css
│   └── app.js
└── test/                  … 動作検証
    ├── engine.test.js     … 役判定＋数千回のランダム対局でチップ保存則を検証
    └── integration.test.js… 実サーバーに複数接続して1ハンドを通す
```

## ローカルでの起動方法

前提：Node.js 18 以上。

```bash
cd ポーカー
npm install
npm start
```

ブラウザで `http://localhost:3000` を開けば動きます。

**同じ Wi-Fi 内の他の端末（スマホなど）から遊ぶ場合**：起動した PC の IP アドレス（例 `192.168.1.5`）を調べ、他の端末のブラウザで `http://192.168.1.5:3000` を開けば入室できます。

## インターネット公開（デプロイ）

WebSocket を使うため、常時起動のサーバーが必要です（Vercel のような静的ホスティングでは動きません）。
[Render](https://render.com) や [Railway](https://railway.app) が簡単でおすすめです。

**Render の例**：
1. このフォルダを GitHub リポジトリにする。
2. Render で「New → Web Service」を選び、そのリポジトリを指定。
3. Build Command: `npm install` ／ Start Command: `npm start`。
4. **環境変数 `ACCESS_CODE` に合い言葉を設定**（下記参照）。
5. デプロイ完了後に発行される URL と合い言葉を身内に共有すれば、どこからでも対戦できます。

ポート番号は環境変数 `PORT` から自動で読み込みます（設定不要）。

### 合い言葉ロック（身内限定）

環境変数 `ACCESS_CODE` を設定すると、サイトを開いた人にまず合い言葉の入力を求めます。
正しい合い言葉を知っている人だけが部屋の作成・参加をできます（サーバー側で検証するため、画面を回避しても入れません）。

- **設定する**：Render の Environment に `ACCESS_CODE = 好きな合い言葉` を追加。
- **変更する**：同じ環境変数を書き換えて再デプロイ。
- **ローカル開発**：`ACCESS_CODE` を設定しなければロックは無効（誰でも入れる）。
  合い言葉付きで試すなら `ACCESS_CODE=あいことば npm start`。

合い言葉はコードには含めず環境変数だけに置くので、GitHub には残りません。

## 今後の編集ポイント

- **色・見た目**：`public/styles.css` の先頭「デザイントークン」（`--gold` などの CSS 変数）を変えると全体の配色が変わります。フェルトの緑は `--felt`。
- **ゲームのデフォルト設定**：初期チップ・ブラインドの初期値は `public/index.html` の入力欄と `src/room.js` の `config` 既定値。
- **ルール挙動**：ベットや配当のロジックは `src/poker/game.js`、役の強さは `src/poker/handEvaluator.js` に集約しています。
